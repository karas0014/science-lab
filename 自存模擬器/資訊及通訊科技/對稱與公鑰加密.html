✓ 解決分發問題：公鑰可公開分發,只有對應的私鑰能解密,毋須事先共用秘密。密鑰越長越難破解。(此處為教學用的極小 RSA)✓ Solves key distribution: the public key can be shared openly; only the matching private key decrypts, so no secret needs pre-sharing. Longer keys are harder to break. (Tiny RSA shown for teaching.)

對稱加解密用同一把密鑰,快但難安全分發。公鑰 (非對稱)用一對鑰匙:公鑰加密、私鑰解密。HTTPS 常先用公鑰安全交換一把對稱密鑰,再用對稱加密傳大量資料。 Symmetric uses one key for both encrypt and decrypt — fast but hard to distribute safely. Public-key (asymmetric) uses a pair: public to encrypt, private to decrypt. HTTPS typically uses public-key to exchange a symmetric key safely, then symmetric encryption for the bulk data.
